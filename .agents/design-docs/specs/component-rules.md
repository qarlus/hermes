# Component Rules

## Buttons

### Icon button
Default action primitive for repeated workspace verbs.

Use for:
- split
- reconnect
- dock/panel toggles
- overflow
- Git quick actions

### Text button
Use sparingly.

Use for:
- connect
- save
- explicit setup
- destructive confirmations
- clarity-heavy decisions

## Pills
Use for compact context only.

Good:
- repo
- branch
- dirty count
- sync state

Bad:
- primary navigation
- large action controls
- decorative labels

## Panels
Should read as attached context, not freestanding cards.

Rules:
- minimal visible borders
- strong internal spacing
- compact headers
- expandable when needed

## Tabs
Editor-style only.

Rules:
- title = identity
- metadata belongs elsewhere
- close on hover
- state via small indicators

## Lists and rows
Rows should feel fast to scan.

Rules:
- hover state is tonal, not glowy
- selection is obvious through fill/contrast
- actions reveal progressively
- avoid always-visible action clutter
